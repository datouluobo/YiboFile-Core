# 可拖动区域布局实现

## 问题解决

### ✅ 1. 修复run.bat每次打开两个程序的问题

**问题原因**：
- `dotnet run`命令会启动应用程序
- 如果编译失败，会再次尝试启动
- 没有检查是否已有实例在运行

**解决方案**：
- 添加实例检查：启动前检查是否已有OoiMRR.exe在运行
- 直接运行编译好的exe文件：避免重复编译和启动
- 改进错误处理：只有在编译失败时才重新编译

**新的run.bat逻辑**：
```batch
1. 检查是否已有实例运行
2. 如果已有实例，提示用户先关闭
3. 尝试直接运行已编译的exe文件
4. 如果没有exe文件，先编译再运行
5. 使用start命令启动，避免阻塞
```

### ✅ 2. 实现可拖动的区域布局

**设计理念**：
将整个界面分成几个相对独立的区域，每个区域都可以通过拖动分割器来调整大小和位置。

## 新的布局结构

### 🏗️ 布局层次
```
DockPanel (主容器)
├── Menu (工具栏 - 固定在顶部)
└── Grid (主内容区域)
    ├── Column 0: 左侧功能栏 (250px, 最小200px)
    ├── Column 1: 垂直分割器1 (4px)
    ├── Column 2: 中间文件浏览器 (*, 最小300px)
    ├── Column 3: 垂直分割器2 (4px)
    ├── Column 4: 右侧预览和备注区 (400px, 最小300px)
    ├── Row 0: 主内容区 (*)
    ├── Row 1: 水平分割器 (4px)
    └── Row 2: 信息区 (150px, 最小100px)
```

### 🎛️ 分割器配置

#### 垂直分割器1 (左侧功能栏 ↔ 中间文件浏览器)
```xml
<GridSplitter Grid.Row="0" Grid.Column="1" Grid.RowSpan="3" 
              Width="4" Background="#DEE2E6" 
              HorizontalAlignment="Center" VerticalAlignment="Stretch"
              ResizeBehavior="PreviousAndNext"/>
```

#### 垂直分割器2 (中间文件浏览器 ↔ 右侧预览区)
```xml
<GridSplitter Grid.Row="0" Grid.Column="3" Grid.RowSpan="3" 
              Width="4" Background="#DEE2E6" 
              HorizontalAlignment="Center" VerticalAlignment="Stretch"
              ResizeBehavior="PreviousAndNext"/>
```

#### 水平分割器 (主内容区 ↔ 信息区)
```xml
<GridSplitter Grid.Row="1" Grid.Column="0" Grid.ColumnSpan="5" 
              Height="4" Background="#DEE2E6" 
              HorizontalAlignment="Stretch" VerticalAlignment="Center"
              ResizeBehavior="PreviousAndNext"/>
```

## 功能特性

### 🎯 可调整区域
1. **左侧功能栏**：可以调整宽度 (200px - 无限制)
2. **中间文件浏览器**：可以调整宽度 (300px - 无限制)
3. **右侧预览和备注区**：可以调整宽度 (300px - 无限制)
4. **底部信息区**：可以调整高度 (100px - 无限制)

### 🎨 视觉设计
- **分割器颜色**：#DEE2E6 (浅灰色)
- **分割器宽度**：4px (垂直) / 4px (水平)
- **最小尺寸限制**：防止区域过小影响使用
- **平滑调整**：拖动时实时调整大小

### 🔧 技术实现

#### GridSplitter属性说明
- `ResizeBehavior="PreviousAndNext"`：调整相邻两个区域的大小
- `HorizontalAlignment="Center"`：水平居中对齐
- `VerticalAlignment="Stretch"`：垂直拉伸填充
- `Grid.RowSpan="3"`：跨越所有行
- `Grid.ColumnSpan="5"`：跨越所有列

#### 最小尺寸限制
- `MinWidth="200"`：左侧功能栏最小宽度
- `MinWidth="300"`：中间和右侧区域最小宽度
- `MinHeight="100"`：信息区最小高度

## 技术优势

### 🔧 实现方式
- **GridSplitter**：WPF内置的分割器控件
- **Grid布局**：灵活的网格布局系统
- **DockPanel**：工具栏固定在顶部
- **最小尺寸**：防止布局破坏

### 🎯 性能优化
- **硬件加速**：WPF的硬件加速渲染
- **平滑动画**：分割器调整时的平滑效果
- **内存效率**：不创建额外的控件实例

### 🔒 稳定性
- **最小尺寸保护**：防止区域过小
- **布局恢复**：窗口大小变化时保持比例
- **错误处理**：分割器操作异常处理

## 使用说明

### 🖱️ 操作方法
1. **调整宽度**：将鼠标悬停在垂直分割器上，出现调整光标时拖动
2. **调整高度**：将鼠标悬停在水平分割器上，出现调整光标时拖动
3. **重置布局**：关闭并重新打开应用程序恢复默认布局

### ⚠️ 注意事项
1. **最小尺寸**：各区域都有最小尺寸限制，无法调整到过小
2. **布局保存**：当前版本不保存布局设置，重启后恢复默认
3. **性能影响**：频繁调整布局可能影响性能
