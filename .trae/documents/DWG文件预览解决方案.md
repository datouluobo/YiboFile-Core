# DWG文件预览解决方案

## 现状分析

* 当前项目使用`CadPreview`类处理DWG/DXF文件预览

* DXF文件可通过内嵌的`DxfRenderEngine`直接渲染

* DWG文件仅能通过外部查看器打开或显示说明页面

* 缺少内置的DWG渲染能力

## 解决方案

### 方案1：集成DWG转DXF转换工具（推荐）

**技术路线**：使用外部转换工具将DWG转换为DXF，再利用现有DXF渲染管道

**实施步骤**：

1. 选择合适的开源转换工具（如ODA File Converter、QCAD命令行工具）
2. 在项目中集成转换工具调用逻辑
3. 扩展`CadPreview`类，添加DWG转DXF的处理分支
4. 实现转换缓存机制，避免重复转换
5. 添加错误处理和进度反馈

**代码修改点**：

* `Previews/CadPreview.cs`：添加DWG转换分支

* `Rendering/DxfRenderEngine.cs`：确保支持转换后的DXF文件

* `Services/CadImageCache.cs`：扩展缓存支持DWG文件

<br />

### 方案3：增强系统集成和用户引导

**技术路线**：优化系统Shell缩略图获取和用户引导

**实施步骤**：

1. 优化`ThumbnailConverter`，增强DWG缩略图获取
2. 改进用户引导页面，提供更清晰的查看器安装指南
3. 添加自动检测和提示功能

**代码修改点**：

* `Controls/Converters/ThumbnailConverter.cs`：优化DWG缩略图获取

* `Previews/CadPreview.cs`：更新说明页面内容

## 实施优先级

1. 方案1：集成DWG转DXF转换工具（核心功能）
2. 方案3：增强系统集成和用户引导（快速改进）

## 测试策略

1. 功能测试：验证各种版本DWG文件的预览效果
2. 性能测试：测试转换和渲染的耗时
3. 兼容性测试：测试不同CAD软件生成的DWG文件
4. 错误处理：测试异常情况下的用户体验

## 预期效果

* DWG文件可以直接在应用内预览，无需外部查看器

* 提供一致的预览体验，与DXF文件保持统一，同时优化DXF文件预览控件、键盘鼠标控制方式

* 支持常见的AutoCAD版本和复杂图元

* 具备良好的错误处理和用户引导

## 风险评估

* 转换工具的兼容性和性能

* 开源转换工具对高版本DWG文件的支持

* 转换过程的耗时和资源占用

## 依赖项

* 外部DWG转换工具

* 可能需要添加新的NuGet包
