# 第一阶段重构详细计划：止血与组织 (Cleanup & Organization)

> 基于 [代码库全面分析报告](CODEBASE_ANALYSIS_REPORT.md) 制定。本阶段目标是降低 `<MainWindow.xaml.cs>` 复杂度，理清文件结构，为后续 MVVM 迁移打好基础，**不涉及**大规模的业务逻辑重写。

## 1. 目标 (Objectives)
- **降低复杂度**: 将 `MainWindow.xaml.cs` (313KB) 拆分为多个管理单元。
- **提高可读性**: 消除文件内部的嵌套类定义。
- **规范结构**: 对 `Services` 文件夹内的 19 个子目录进行归类整理。
- **无破坏性**: 确保重构过程中不破坏现有功能 (Refactor without behavior change)。

## 2. 详细执行步骤 (Execution Steps)

### 2.1. 提取嵌套类 (Extract Nested Types)
目前 `MainWindow.xaml.cs` 中可能定义了大量的辅助类、Struct 或 Enum。需要将它们移动到独立的文件中。

- [ ] **扫描**: 找出 `MainWindow` 类内部定义的所有 `class`, `struct`, `enum`。
- [ ] **移动**:
  - 模型类 -> 移至 `Models/` 目录 (或新建 `Models/Internal/`)。
  - 辅助逻辑 -> 移至 `Services/TypeHelpers/` 或相关目录。
  - 仅 UI 相关的辅助类 -> 移至 `UI/Helpers/`。

### 2.2. 拆分 MainWindow (Split MainWindow)
使用 `partial class` 技术，按功能领域将 `MainWindow.xaml.cs` 物理拆分为多个文件。这不是最终的架构解决方案，但是通往 MVVM 的必经之路（先拆分，再逐个迁移）。

建议拆分方案：
- [ ] **`MainWindow.xaml.cs`**: 保留构造函数、初始化逻辑、生命周期事件。
- [ ] **`MainWindow.Events.cs`**: 存放通用的 UI 事件处理 (Button Click, Loaded 等)。
- [ ] **`MainWindow.Menu.cs`**: 存放顶部菜单栏、上下文菜单相关逻辑。
- [ ] **`MainWindow.DragDrop.cs`**: 存放与拖拽相关的逻辑（作为 `DragDropManager` 的桥接）。
- [ ] **`MainWindow.Navigation.cs`**: 存放文件导航、路径跳转逻辑。
- [ ] **`MainWindow.Preview.cs`**: 存放预览面板相关逻辑。
- [ ] **`MainWindow.Search.cs`**: 存放搜索栏相关逻辑。

*注：拆分后，虽然代码量总量没变，但每个文件的认知负载将大幅降低。*

### 2.3. 整理 Services 目录 (Organize Services)
目前 `Services` 目录过于扁平且混乱。需要创建一级分类并将现有服务归档。

建议目录结构调整：
```text
Services/
├── Core/               # 核心基础服务 (Database, Config, Logging)
├── FileSystem/         # 文件操作相关 (FileOperations, FileInfo, FileList)
├── Media/              # 媒体处理 (FFmpeg, Magick, CAD, Previews)
├── UI/                 # UI 交互服务 (Dialogs, DragDrop, Layout, Windows)
├── Search/             # 搜索相关 (Everything, Search strategies)
└── Features/           # 特定业务功能 (TagTrain, Favorites, Tags)
```

- [ ] **创建目录**: 建立上述子目录。
- [ ] **移动文件**: 将现有的 19 个文件夹和 13 个根文件归类到上述目录中。
- [ ] **修复命名空间**: *（可选，建议暂缓）* 移动文件后暂不修改命名空间，以避免大规模破坏引用。仅物理移动，并在 `csproj` 确保编译通过。

## 3. 验证标准 (Verification)
- [ ] **编译通过**: 所有修改后，项目必须能成功编译 (Build Success)。
- [ ] **启动正常**: 应用程序能正常启动，无运行时崩溃。
- [ ] **功能冒烟测试**:
  - 打开文件夹/导航正常。
  - 搜索功能正常。
  - 图片/视频预览正常。

## 4. 下一步预览
完成本阶段后，我们将拥有一个更清晰的文件结构，接下来可以进入 **第二阶段：MVVM 渐进式采用**，开始将 `MainWindow.Events.cs` 等文件中的逻辑剥离到 ViewModel 中。
