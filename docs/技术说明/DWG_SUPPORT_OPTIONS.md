# DWG 文件支持方案对比

## 当前方案：ODA File Converter (CLI)
- **优点**：免费、转换质量高
- **缺点**：需要用户单独安装
- **状态**：已实现

## 替代方案

### 方案 A：自动下载 ODA File Converter（推荐）
**实现思路**：
1. 程序首次运行时检测是否安装 ODA
2. 如果未安装，提示用户一键下载安装
3. 下载后自动解压到 `Dependencies/ODAFileConverter/`

**优点**：
- 用户体验好，一键安装
- 合法合规
- 转换质量高

**缺点**：
- 首次使用需要下载（约 100MB）

### 方案 B：仅支持 DXF，放弃 DWG
**实现思路**：
- 移除 DWG 转换功能
- 只支持 DXF 文件预览（已完美支持）
- 提示用户使用 AutoCAD 等软件将 DWG 另存为 DXF

**优点**：
- 无需外部依赖
- 代码简单

**缺点**：
- 不支持 DWG 文件

### 方案 C：使用商业库（付费）
**可选库**：
- **CadLib** (Wout Ware) - 约 $500
- **AnyCAD.Net** - 约 $300
- **Aspose.CAD** - 约 $1000/年

**优点**：
- 原生支持 DWG
- 功能强大

**缺点**：
- 需要付费
- 增加程序体积

### 方案 D：使用开源库 netDxf（仅 DXF）
**库**：`netDxf` (MIT License)

**优点**：
- 完全免费开源
- 可以直接集成
- 支持读写 DXF

**缺点**：
- 不支持 DWG
- 功能相对简单

## 推荐方案：A（自动下载 ODA）

我建议实现方案 A，为用户提供一键下载 ODA File Converter 的功能。这样既合法合规，又能提供良好的用户体验。
