# FileSystemWatcherService 测试指南

## 手动测试步骤

### 1. 基本功能测试

#### 测试文件创建
1. 启动应用程序
2. 导航到任意文件夹（如桌面）
3. 在文件资源管理器中，在该文件夹内创建一个新文件（如 `test.txt`）
4. **预期结果**：等待约3秒后，应用程序中的文件列表自动刷新，新文件出现

#### 测试文件删除
1. 在应用程序中选中一个文件
2. 在文件资源管理器中删除该文件
3. **预期结果**：等待约3秒后，文件从列表中消失

#### 测试文件重命名
1. 在文件资源管理器中重命名一个文件
2. **预期结果**：等待约3秒后，文件列表更新，显示新名称

#### 测试文件夹创建/删除
1. 在文件资源管理器中创建或删除文件夹
2. **预期结果**：等待约3秒后，文件夹列表自动更新

### 2. 防抖机制测试

#### 测试防抖延迟
1. 快速连续创建多个文件（如5个文件，间隔1秒）
2. **预期结果**：
   - 不会立即刷新
   - 停止操作后约3秒，一次性刷新所有变化
   - 控制台输出：`[FileSystemWatcherService] 自动刷新文件列表...`

#### 测试防抖重置
1. 创建文件A，等待2秒
2. 创建文件B
3. **预期结果**：防抖定时器重置，从创建文件B时重新计时3秒

### 3. 路径切换测试

#### 测试切换监控路径
1. 在文件夹A中创建文件
2. 切换到文件夹B
3. 在文件夹A中创建文件
4. **预期结果**：文件夹B不会收到文件夹A的变化通知

### 4. 异常情况测试

#### 测试监控不存在的路径
1. 尝试监控一个不存在的路径
2. **预期结果**：不会报错，静默失败

#### 测试监控已删除的路径
1. 监控一个文件夹
2. 删除该文件夹
3. **预期结果**：不会报错，后续变化不会触发事件

## 调试输出验证

### 如何查看调试输出

**Visual Studio:**
1. 运行应用程序（F5）
2. 打开"输出"窗口：`视图` → `输出` 或按 `Ctrl+Alt+O`
3. 在"显示输出来源"下拉框中选择"调试"

**预期调试输出：**
- `[MainWindow] 文件系统变化: Created - test.txt` - 文件系统变化事件触发时
- `[FileSystemWatcherService] 自动刷新文件列表...` - 防抖定时器触发时
- `[MainWindow] 收到刷新请求，开始刷新文件列表...` - 开始刷新时
- `[MainWindow] 正在加载中，跳过刷新请求` - 如果正在加载时收到请求
- `[FileSystemWatcherService] 无法设置文件监视器: ...` - 设置失败时（如果有）

### 测试调试输出

1. 启动应用程序（调试模式）
2. 打开输出窗口
3. 导航到一个文件夹
4. 在文件资源管理器中创建文件
5. **验证输出**：应该看到文件系统变化事件，然后3秒后看到刷新请求

## 快速测试脚本

### PowerShell 测试脚本

在测试文件夹中运行以下 PowerShell 脚本，快速创建/删除文件：

```powershell
# 创建测试文件
1..5 | ForEach-Object {
    New-Item -Path "test_$_.txt" -ItemType File -Force
    Start-Sleep -Seconds 1
}

# 等待防抖触发（约3秒）
Start-Sleep -Seconds 5

# 删除测试文件
1..5 | ForEach-Object {
    Remove-Item -Path "test_$_.txt" -Force
    Start-Sleep -Seconds 1
}
```

**预期行为：**
- 创建5个文件后，等待约3秒，应用程序自动刷新
- 删除5个文件后，等待约3秒，应用程序自动刷新
- 总共只刷新2次，而不是10次（证明防抖有效）

## 常见问题排查

### 问题1：文件变化后没有自动刷新

**检查项：**
1. 确认输出窗口有 `[MainWindow] 文件系统变化` 日志
2. 确认3秒后有 `[FileSystemWatcherService] 自动刷新文件列表...` 日志
3. 检查 `_isLoadingFiles` 是否为 false（如果为 true，会跳过刷新）

### 问题2：刷新过于频繁

**检查项：**
1. 确认防抖定时器正常工作（应该看到3秒延迟）
2. 检查是否有多个 FileSystemWatcher 实例

### 问题3：切换路径后仍监控旧路径

**检查项：**
1. 确认 `SetupFileWatcher()` 在路径切换时被调用
2. 检查 `IsWatching()` 方法返回是否正确
